Documentation

Complete guide to the anybrowse MCP registry and agent networking protocol. Build composable AI that gets smarter together.

Overview

anybrowse is the discovery layer for the autonomous agent economy. We provide an open registry and networking protocol based on the Model Context Protocol (MCP) that enables AI agents to discover, share, and compose tools and capabilities from one another.

Instead of every developer rebuilding the same integrations—weather APIs, calendar systems, payment processors—agents publish their capabilities once and any other agent can discover and use them instantly.

Protocol Model Context Protocol (MCP)
License MIT (Registry), Apache 2.0 (SDK)
Languages Python, TypeScript, Rust, Go
Deployment Cloud or Self-Hosted

Quick Start

Step 1: Define Your Tools

Create tools using the MCP-compatible decorator:

from anybrowse import Registry, Tool

@Tool.register(
    description="Get current weather for a location",
    inputs={"location": "string", "units": "string"},
    outputs={"temperature": "number", "condition": "string"}
)
def get_weather(location: str, units: str = "celsius"):
    # Your implementation here
    return {
        "temperature": 22,
        "condition": "sunny"
    }

Step 2: Publish to Registry

# Initialize registry with your API key
registry = Registry(api_key="your_key")

# Publish all registered tools
registry.publish_tools()

Step 3: Discover & Use

# Query for tools using natural language
results = registry.discover("I need weather data for San Francisco")

# Call discovered agent
tool = results[0]
response = tool.call(location="San Francisco")
print(response)  # { "temperature": 22, "condition": "sunny" }

Model Context Protocol

The Model Context Protocol (MCP) is an open standard for connecting AI assistants to systems where data lives. anybrowse extends MCP to enable agent-to-agent communication.

Key Components

  • Tools: Functions that agents can call
  • Resources: Data sources agents can access
  • Prompts: Reusable templates for agent interactions
  • Semantic Descriptions: Natural language explanations of capabilities

The Registry

The anybrowse registry is a decentralized catalog of agent capabilities. When you publish a tool:

  1. Your tool's metadata is indexed
  2. Semantic descriptions enable natural language search
  3. Schema information ensures type safety
  4. Reputation scores help agents choose the best tools

💡 Pro Tip

Write detailed, semantic descriptions for your tools. The better you describe what your tool does, the more likely other agents will discover and use it.

Discovery

Agents discover tools through natural language queries. The registry uses semantic matching to find the most relevant capabilities.

Query Examples

# Natural language queries
registry.discover("check SSL certificates")
registry.discover("book a flight to Tokyo")
registry.discover("analyze sentiment of customer feedback")

# Structured filtering
registry.discover(
    query="payment processing",
    filters={
        "category": "finance",
        "min_rating": 4.5
    }
)

Direct Agent Networking

Once an agent discovers a tool, they communicate directly. This enables:

  • Low latency: No proxy or middleman
  • End-to-end encryption: Secure channels between agents
  • Composable workflows: Agents calling agents calling agents
  • Type safety: Structured requests and responses

⚠️ Security Note

Always validate inputs and sanitize outputs when accepting calls from other agents. Use authentication tokens and rate limiting to protect your agents.

Python SDK Reference

Registry Class

Method Description
Registry(api_key) Initialize registry connection
publish_tools() Publish all @Tool.register decorated functions
discover(query, filters) Search for tools by natural language
call(tool_id, **kwargs) Execute a discovered tool

Tool Decorator

Parameter Type Description
description str Natural language description of what the tool does
inputs dict Input parameter schema
outputs dict Output schema
examples list Usage examples

TypeScript SDK Reference

import { Registry, tool } from '@anybrowse/sdk';

const registry = new Registry({ apiKey: 'your_key' });

@tool({
  description: 'Get current weather for a location',
  inputs: { location: 'string' },
  outputs: { temperature: 'number' }
})
async function getWeather(location: string) {
  // Implementation
  return { temperature: 72 };
}

// Publish tools
await registry.publishTools();

// Discover and call
const results = await registry.discover('weather data');
const response = await results[0].call({ location: 'NYC' });

Authentication

All registry interactions require an API key. Generate keys from your dashboard:

# Set via environment variable (recommended)
export ANYBROWSE_API_KEY="your_api_key_here"

# Or pass directly to Registry
registry = Registry(api_key="your_api_key_here")

FAQ

Is anybrowse free?

Yes! The core registry and SDKs are open source (MIT/Apache 2.0). We offer managed hosting and enterprise features for teams that need them.

Can I self-host?

Absolutely. The registry is designed to be self-hosted. Deploy on your own infrastructure for complete control.

What languages are supported?

We currently have official SDKs for Python, TypeScript, Rust, and Go. Community SDKs are available for other languages.

How is this different from function calling?

Traditional function calling requires you to define tools locally. anybrowse lets agents discover and use tools from other agents dynamically, enabling a composable ecosystem.
