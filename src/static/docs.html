API Reference

AnyBrowse.dev uses the x402 protocol for micropayments. All endpoints require payment via USDC on Base network. No API keys needed—your wallet is your authentication.

POST /scrape

Convert a single URL to clean, LLM-ready Markdown using a real Chrome browser.

Request Body

Parameter Type Description
url required string The URL to scrape (must start with http:// or https://)
waitFor optional number Milliseconds to wait for JavaScript rendering (default: 5000)
timeout optional number Maximum time to wait for page load in ms (default: 30000)

Example Request

curl
curl -X POST https://anybrowse.dev/scrape \\
  -H "Content-Type: application/json" \\
  -H "X-Payment: $SIGNED_PAYMENT" \\
  -d '{"url": "https://example.com", "waitFor": 3000}'

Example Response

200 OK
# Example Domain

This domain is for use in illustrative examples in documents. You may use this domain in literature without prior coordination or asking for permission.

[More information...](https://www.iana.org/domains/example)
Payment Required

If no valid payment header is provided, the endpoint returns 402 Payment Required with payment requirements in the response headers.

POST /crawl

Multi-page crawling with recursive link following. Ideal for documentation sites and sitemaps.

Request Body

Parameter Type Description
url required string Starting URL for the crawl
maxPages optional number Maximum pages to crawl (default: 10, max: 100)
maxDepth optional number Maximum link depth from starting URL (default: 2)
sameDomain optional boolean Only crawl pages on the same domain (default: true)

Example Response

200 OK - application/json
{
  "pages": [
    {
      "url": "https://example.com",
      "markdown": "# Example Domain...",
      "title": "Example Domain"
    },
    {
      "url": "https://example.com/about",
      "markdown": "# About Us...",
      "title": "About"
    }
  ],
  "crawled": 2,
  "skipped": 0
}
GET /mcp

Model Context Protocol (MCP) server endpoint for AI agent integration. Supports tool discovery and execution.

Available Tools

Tool Description
scrape Convert any URL to clean, LLM-optimized Markdown using real Chrome browsers

MCP Tool Schema

scrape tool schema
{
  "name": "scrape",
  "description": "Convert any URL to clean, LLM-optimized Markdown. Uses real Chrome browsers with JavaScript execution.",
  "inputSchema": {
    "type": "object",
    "properties": {
      "url": {
        "type": "string",
        "description": "The URL to scrape (must start with http:// or https://)"
      }
    },
    "required": ["url"]
  }
}

Error Codes

Status Code Description
200 OK Successful scrape/crawl
400 BAD_REQUEST Invalid URL or missing required parameters
402 PAYMENT_REQUIRED Payment required. See X-Payment-Requirements header
403 FORBIDDEN URL blocked (private IP, localhost, etc.)
404 NOT_FOUND Target URL returned 404
429 TOO_MANY_REQUESTS Rate limit exceeded
500 INTERNAL_ERROR Browser or server error
504 TIMEOUT Page load timeout

Getting Started

AnyBrowse uses the x402 protocol for seamless micropayments. Follow these steps to start scraping autonomously.

1. Setup USDC Wallet on Base

1

Get a Web3 Wallet

Install Coinbase Wallet, MetaMask, or any EVM-compatible wallet.

2

Add Base Network

Add Base mainnet to your wallet:

  • Network Name: Base
  • RPC URL: https://mainnet.base.org
  • Chain ID: 8453
  • Currency Symbol: ETH
3

Get USDC on Base

Bridge USDC from Ethereum or buy directly on Coinbase. You'll need a small amount of ETH for gas.

2. Configure x402 Client

Install an x402-compatible client to handle payments automatically.

npm install
npm install x402-fetch
TypeScript/JavaScript
import { x402Fetch } from 'x402-fetch';

const client = x402Fetch({
  wallet: {
    privateKey: process.env.WALLET_PRIVATE_KEY,
    network: 'base'
  },
  facilitator: 'coinbase' // or 'cloudflare'
});

3. Make Your First API Call

JavaScript
const response = await client('https://anybrowse.dev/scrape', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ 
    url: 'https://example.com',
    waitFor: 3000 
  })
});

const markdown = await response.text();
console.log(markdown);
Test on Base Sepolia

Use network: 'base-sepolia' for testing. Get test USDC from the Circle faucet.

4. Handle Payment Responses

Understand how x402 payments work:

Payment Flow (automatic with x402-fetch)
1. Client → POST /scrape (no payment)
2. Server → 402 Payment Required
   X-Payment-Requirements: {"amount": "2000", "asset": "USDC", ...}
3. Client → Signs EIP-3009 authorization (off-chain, gasless)
4. Client → POST /scrape + X-Payment: <signed-auth>
5. Server → Verifies with facilitator
6. Server → Returns scraped Markdown

MCP Integration

AnyBrowse provides a Model Context Protocol (MCP) server for seamless integration with Claude, Cursor, Windsurf, and other MCP-compatible tools.

Install MCP Server

npm (global)
npm install -g @anybrowse/mcp

Configure Claude Desktop

Add to your Claude Desktop configuration:

~/Library/Application Support/Claude/claude_desktop_config.json
{
  "mcpServers": {
    "anybrowse": {
      "command": "npx",
      "args": ["-y", "@anybrowse/mcp"],
      "env": {
        "X402_WALLET_PRIVATE_KEY": "0x...",
        "X402_NETWORK": "base"
      }
    }
  }
}

Configure Cursor

In Cursor, go to Settings → AI → MCP Servers and add:

Cursor MCP Config
Name: anybrowse
Type: command
Command: npx -y @anybrowse/mcp

Environment Variables:
  X402_WALLET_PRIVATE_KEY=0x...
  X402_NETWORK=base

Configure Windsurf

Add to your Windsurf MCP configuration:

~/.codeium/windsurf/mcp_config.json
{
  "mcpServers": {
    "anybrowse": {
      "command": "npx",
      "args": ["-y", "@anybrowse/mcp"],
      "env": {
        "X402_WALLET_PRIVATE_KEY": "0x..."
      }
    }
  }
}

Use Scrape Tool in Conversations

Once configured, simply ask your AI assistant:

"Scrape https://example.com and summarize the content for me."

The AI will automatically call the scrape tool, handle the x402 payment, and return the Markdown content—all in the conversation flow.

Best Practices

Rate Limiting

To ensure fair usage and system stability:

  • Scrape: 60 requests per minute per wallet
  • Crawl: 10 requests per minute per wallet
  • Max page size: 10MB per page
  • Max execution time: 60 seconds per request

If you hit rate limits, you'll receive a 429 Too Many Requests response with a Retry-After header.

Error Handling

Robust error handling example
try {
  const response = await client('https://anybrowse.dev/scrape', {
    method: 'POST',
    body: JSON.stringify({ url })
  });
  
  if (response.status === 402) {
    // x402 client should handle this automatically
    console.error('Payment failed - check wallet balance');
  } else if (response.status === 429) {
    const retryAfter = response.headers.get('Retry-After');
    console.log(`Rate limited. Retry after ${retryAfter}s`);
  } else if (!response.ok) {
    throw new Error(`HTTP ${response.status}: ${await response.text()}`);
  }
  
  const markdown = await response.text();
} catch (error) {
  console.error('Scrape failed:', error);
}

Cost Optimization

Cost-Saving Tips
  • Use crawl for multiple pages: More efficient than individual scrape calls
  • Set appropriate waitFor: Don't wait longer than necessary for JS rendering
  • Cache results: Store scraped content locally to avoid duplicate charges
  • Use sameDomain: Prevents accidental crawling of external links
  • Test on Sepolia first: Validate your integration with testnet before mainnet
