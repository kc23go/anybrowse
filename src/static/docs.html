API REFERENCE

All endpoints accept JSON and return Markdown or JSON. Authenticate with an API key (Pro) or x402 payment header. Free tier: 10 requests/day, no auth required.

POST /scrape

Convert a single URL to clean, LLM-ready Markdown. Uses a real Chrome browser with stealth mode and automatic proxy fallback for blocked sites.

Request body

ParameterTypeDescription
url requiredstringThe URL to scrape. Must start with http:// or https://
waitFor optionalnumberMilliseconds to wait for JS rendering (default: 5000)
timeout optionalnumberMax page load time in ms (default: 30000)

Example

curl
curl -X POST https://anybrowse.dev/scrape \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com"}'
With API key (Pro)
curl -X POST https://anybrowse.dev/scrape \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ab_your_key_here" \
  -d '{"url": "https://example.com", "waitFor": 3000}'

Response (200 OK)

Plain Markdown text
# Example Domain

This domain is for use in illustrative examples in documents.
You may use this domain in literature without prior coordination.

[More information...](https://www.iana.org/domains/example)
POST /crawl

Multi-page crawling with recursive link following. Ideal for documentation sites, product pages, and sitemaps.

Request body

ParameterTypeDescription
url requiredstringStarting URL for the crawl
maxPages optionalnumberMax pages to crawl (default: 10, max: 100)
maxDepth optionalnumberMax link depth from start URL (default: 2)
sameDomain optionalbooleanOnly follow same-domain links (default: true)

Example response

200 OK — application/json
{
  "pages": [
    {
      "url": "https://example.com",
      "title": "Example Domain",
      "markdown": "# Example Domain..."
    }
  ],
  "crawled": 2,
  "skipped": 0
}
POST /serp/search

Run a Google search and return results as structured data or Markdown. Useful for agents that need to find and then scrape relevant pages.

Request body

ParameterTypeDescription
query requiredstringSearch query
num optionalnumberNumber of results to return (default: 10)
GET /mcp

Streamable-HTTP MCP server. Exposes scrape, crawl, and search tools to any MCP-compatible client.

Available tools

ToolDescription
scrapeConvert any URL to LLM-ready Markdown
crawlMulti-page crawl, returns array of pages
searchGoogle search, returns structured results

Error codes

All error responses include a JSON body with a machine-readable reason field.

Error response shape
{
  "error": "Human-readable description",
  "reason": "REASON_CODE",
  "upgrade": "https://anybrowse.dev/checkout"
}

Payment flow

x402 payments are automatic with the x402-fetch library. Here is what happens under the hood:

x402 flow
1. Client  POST /scrape (no payment header)
2. Server  402 Payment Required
           X-Payment-Requirements: {"amount": "2000", "asset": "USDC", ...}
3. Client  Signs EIP-3009 authorization (off-chain, gasless)
4. Client  POST /scrape + X-Payment: <signed-auth>
5. Server  Verifies with Coinbase facilitator
6. Server  Returns scraped Markdown

Error handling

Robust error handling
try {
  const response = await client('https://anybrowse.dev/scrape', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ url })
  });

  if (response.status === 402) {
    // x402 client handles this automatically
    // Manual fallback: check wallet balance
    console.error('Payment failed');
  } else if (response.status === 429) {
    const retryAfter = response.headers.get('Retry-After');
    await sleep(retryAfter * 1000);
    // retry...
  } else if (!response.ok) {
    const err = await response.json();
    // err.reason: TIMEOUT | BLOCKED | NOT_FOUND | INTERNAL_ERROR
    if (err.reason === 'TIMEOUT') {
      // Page was slow — increase waitFor or skip
    } else if (err.reason === 'BLOCKED') {
      // Site blocked scraping after proxy fallback
    }
    throw new Error(`[${err.reason}] ${err.error}`);
  }

  return await response.text();
} catch (err) {
  console.error('Scrape failed:', err);
}

Cost tips

Reduce costs

A few things that add up:

  • Use /crawl for multi-page jobs rather than many individual /scrape calls
  • Cache results locally to avoid scraping the same URL twice
  • Set a realistic waitFor value. 2000ms is often enough for most sites
  • Use sameDomain: true to avoid accidentally crawling external links
  • Test on Base Sepolia to validate your integration before spending mainnet USDC
